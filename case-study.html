Portfolio Case Study

ResearchAI - AI SaaS research workspace

ResearchAI turns a user question into a structured professional report with account-based usage, saved history, exports, subscriptions, and production-minded billing flows.

Live demo GitHub available on request Contact

Problem

Many AI tools produce open-ended chat responses that are difficult to save, compare, export, or trust as professional work. ResearchAI explores how an AI product can feel more like a structured research workspace than a prompt box.

Solution

The product guides users from a research question to a document-style report with executive summary, findings, risks, recommendations, source guidance, confidence notes, saved reports, and export options.

Target users

  • Founders validating startup ideas
  • Freelancers and consultants preparing research drafts
  • Operators comparing tools, markets, or strategies
  • Students and self-directed learners structuring plans

Main features

  • Dynamic AI report generation with local fallback
  • Saved reports, pinned reports, history, and exports
  • Clerk authentication and account state
  • Stripe Checkout, Billing Portal, and subscription sync

Technology stack

HTML CSS JavaScript Vercel Functions Clerk Stripe Supabase Gemini API

Architecture overview

The frontend is a framework-free web app. API routes handle secure generation, authentication checks, billing actions, Stripe webhooks, Supabase persistence, and provider integration. The browser never receives server-side secrets.

Frontend UI Vercel API routes Clerk auth Gemini provider Supabase data layer Stripe billing

Security and production work

  • Server-side Clerk token verification for protected API routes.
  • Stripe webhook signature verification using raw request bodies.
  • Server-side usage enforcement for Free and Pro plans.
  • Supabase RLS documentation and service-role-only API access.
  • Content Security Policy and production security headers for Vercel.

Key engineering challenges solved

  • Reliable Pro activation across Stripe, Clerk, and Supabase.
  • Billing Portal cancellation sync when Stripe webhooks and dashboard state differ.
  • Automatic billing status refresh after returning from Stripe Portal.
  • Safe report rendering with escaped user-controlled content.
  • Professional report UX without changing the underlying AI provider architecture.
