# Growth Strategy

Version: 1.0

Project: ResearchAI

Status: Active

---

# Purpose

Growth is not about getting more visitors.

Growth is about creating more value.

More value creates more satisfied users.

Satisfied users create sustainable growth.

ResearchAI should never optimize for vanity metrics.

We optimize for meaningful usage.

---

# North Star Metric

The primary metric of ResearchAI is:

Hours of work saved.

Everything else is secondary.

If users save meaningful time...

The product is improving.

---

# Stage 1

## Validate

Goal:

Confirm that people understand the product.

Questions:

Do users understand what ResearchAI does?

Can they generate their first report without help?

Would they use it again?

Can they explain it to someone else?

Success:

People naturally understand the value.

---

# Stage 2

## Retention

Goal:

Users return voluntarily.

We do not measure success by one visit.

We measure success by repeat usage.

Questions:

Did users come back?

Did they create another report?

Did they trust previous reports?

---

# Stage 3

## Recommendation

Goal:

Users recommend ResearchAI.

The strongest marketing is genuine recommendation.

People should naturally say:

"You should try ResearchAI."

Not because of rewards.

Because it genuinely helped them.

---

# Stage 4

## Revenue

Money is proof of value.

People pay because:

The product saves time.

The product reduces uncertainty.

The product helps make decisions.

Revenue follows value.

Never the opposite.

---

# First 100 Users

The goal is not growth.

The goal is learning.

Every early user is more valuable than hundreds of anonymous visitors.

Collect feedback.

Observe behavior.

Improve quickly.

---

# Product Decisions

Never build features based on guesses.

Build features because:

Users asked.

Users struggled.

Users repeatedly encountered the same problem.

Patterns matter more than opinions.

---

# Feedback

Every piece of feedback should answer one question:

What prevented the user from completing their work?

Ignore cosmetic requests until core problems are solved.

---

# Metrics That Matter

Time to first report.

Report completion rate.

Return rate.

Exports per report.

Average reports per user.

Time saved.

User satisfaction.

Recommendation rate.

---

# Metrics That Do Not Matter

Total page views.

Social media likes.

Random traffic.

Downloads.

Followers.

These numbers only matter if they lead to real usage.

---

# Continuous Improvement

Every release should improve at least one measurable outcome.

Examples:

Faster report generation.

Higher completion rate.

More exports.

Better retention.

If nothing improves...

The release failed.

---

# Growth Philosophy

Growth is earned.

Never forced.

Never manipulated.

Never bought through misleading promises.

Trust creates sustainable growth.

---

# Community

Users should feel they are helping shape ResearchAI.

Listen carefully.

Respond respectfully.

Build transparently.

---

# Marketing

Marketing explains value.

Marketing does not create value.

The product creates value.

Marketing communicates it.

Never confuse the two.

---

# Competition

We do not copy competitors.

We study them.

We understand why users choose them.

Then we solve problems differently.

---

# Success

Growth is successful when users achieve more with less effort.

Not when charts simply go up.

---

# Long-Term Goal

ResearchAI should become part of people's workflow.

Not because they are forced to use it.

Because they cannot imagine doing the work without it.

---

# Final Principle

Every improvement should answer one question:

Does this help users achieve meaningful work faster?

If the answer is yes...

Build it.

If the answer is no...

Do not.
