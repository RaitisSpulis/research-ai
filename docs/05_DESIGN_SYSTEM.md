# Design System

## Purpose

This document defines the visual language of ResearchAI.

Every interface decision must strengthen trust, clarity and professionalism.

The design should never exist to impress.

The design exists to help people complete meaningful work.

---

# Design Philosophy

ResearchAI is not entertainment.

ResearchAI is not social media.

ResearchAI is not a game.

ResearchAI is professional software.

Every screen should feel calm.

Every screen should feel intentional.

Every screen should communicate confidence.

---

# First Impression

Within the first five seconds users should feel:

Professional.

Modern.

Fast.

Trustworthy.

Premium.

Calm.

Never:

Overwhelming.

Noisy.

Confusing.

Cheap.

---

# Visual Identity

ResearchAI should look like software built for professionals.

Think about:

Apple

Stripe

Linear

Notion

Perplexity

OpenAI

Never copy them.

Understand why they feel trustworthy.

Then create our own identity.

---

# Simplicity

If a component can be simplified...

Simplify it.

If an animation can be removed without reducing clarity...

Remove it.

Less is usually better.

---

# Layout

Every page should follow the same visual rhythm.

Large spacing.

Clear hierarchy.

Consistent alignment.

Predictable navigation.

Generous whitespace.

The interface should breathe.

---

# Typography

Typography is the primary communication tool.

Titles should immediately explain purpose.

Paragraphs should remain short.

Avoid large blocks of text.

Prefer visual hierarchy over decoration.

Reading should feel effortless.

---

# Color Philosophy

Colors communicate meaning.

Primary accent:

Action.

Success:

Completion.

Warning:

Attention.

Danger:

Errors only.

Never use color for decoration.

Every color must have meaning.

---

# Dark Theme

Dark mode is the default experience.

It should feel elegant.

Not aggressive.

Backgrounds should use multiple depth levels.

Cards should separate naturally.

Contrast should remain comfortable during long reading sessions.

---

# Cards

Cards are the foundation of the interface.

Every important piece of information lives inside a card.

Cards should feel:

Organized.

Readable.

Structured.

Premium.

Never overcrowded.

---

# Buttons

Every button answers one question:

What happens next?

Primary button:

Main action.

Secondary button:

Alternative action.

Danger button:

Irreversible action.

Buttons should never compete equally.

The primary action must always be obvious.

---

# Icons

Icons support understanding.

They never replace text.

Every icon should have a clear purpose.

Consistency matters more than creativity.

---

# Animations

Animations communicate state.

Not decoration.

Good animations:

Explain movement.

Reduce confusion.

Guide attention.

Bad animations:

Distract.

Delay interaction.

Exist without purpose.

Every animation should improve understanding.

---

# Motion

Motion should feel natural.

Fast.

Smooth.

Confident.

Never dramatic.

Micro-interactions should reward interaction without demanding attention.

---

# Data Visualization

Charts should communicate decisions.

Not decoration.

Every graph should answer a question.

Every number should support a conclusion.

Never show meaningless statistics.

---

# Reports

Reports are premium products.

Each section should feel like it belongs in a consulting document.

Reports should use:

Cards.

Charts.

Tables.

Highlights.

Recommendations.

Confidence indicators.

Action plans.

Professional spacing.

---

# Dashboard

The dashboard is the user's workspace.

It should answer:

What should I continue?

What should I create?

What changed?

What matters today?

Never become a collection of random widgets.

---

# Landing Page

The landing page sells outcomes.

Never technology.

The visitor should immediately understand:

What they gain.

Why it matters.

What happens after clicking.

---

# Consistency

Consistency creates trust.

Spacing.

Corners.

Buttons.

Shadows.

Animations.

Typography.

Everything should follow one system.

---

# Accessibility

Beauty without accessibility is failure.

Readable contrast.

Keyboard navigation.

Visible focus states.

Screen reader compatibility.

Large touch targets.

Everyone should be able to use ResearchAI.

---

# Responsive Design

Desktop.

Tablet.

Mobile.

All three experiences deserve equal attention.

The interface should adapt naturally.

Never simply shrink desktop layouts.

---

# Premium Feeling

Premium is not expensive.

Premium is attention to detail.

Small improvements create trust.

Consistent spacing.

Smooth interactions.

Fast loading.

Clear hierarchy.

These details define quality.

---

# Never Do

Never copy another product.

Never overload screens.

Never use random colors.

Never use inconsistent spacing.

Never add animations for decoration.

Never sacrifice usability for visual effects.

Never confuse users.

---

# Design Goal

The interface should disappear.

The user's work should become the focus.

If users admire the interface for a few seconds...

Good.

If users forget the interface because they are focused on completing work...

Perfect.
