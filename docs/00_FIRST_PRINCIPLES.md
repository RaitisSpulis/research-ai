# ResearchAI First Principles

Version: 1.0

Status: Permanent

Priority: Highest

Read this document before reading any other documentation.

---

# Purpose

Technology changes.

Markets change.

AI models change.

Programming languages change.

These principles do not.

Everything built inside ResearchAI must follow these principles.

If an idea violates them...

The idea is wrong.

---

# Principle 1

We build products.

Not demos.

Everything we create should move closer to becoming a real company.

Never optimize for looking impressive.

Optimize for creating value.

---

# Principle 2

We sell outcomes.

Not AI.

Users never buy artificial intelligence.

Users buy:

Saved time.

Better decisions.

Completed work.

Confidence.

Every feature must strengthen one of these outcomes.

---

# Principle 3

The user is the hero.

ResearchAI is only the guide.

Never build features that make the AI look intelligent.

Build features that make the user more capable.

---

# Principle 4

Trust is our greatest asset.

Trust takes years to build.

Minutes to lose.

Never fake information.

Never invent sources.

Never manipulate users.

Never exaggerate.

When uncertain...

Say so.

---

# Principle 5

Time is the only resource users never recover.

Every click.

Every screen.

Every paragraph.

Every interaction must respect the user's time.

If something wastes time...

Remove it.

---

# Principle 6

Complexity is failure.

If a feature requires explanation...

Improve the feature.

Not the explanation.

Good products feel obvious.

---

# Principle 7

Beauty supports function.

Beauty is never the goal.

Professionalism.

Clarity.

Speed.

Consistency.

These create beautiful products.

---

# Principle 8

Every feature must justify its existence.

Before adding anything ask:

What problem does this solve?

If no clear answer exists...

Do not build it.

---

# Principle 9

Never chase trends.

Today's trend becomes tomorrow's legacy code.

Build timeless systems.

Not fashionable features.

---

# Principle 10

Always think in systems.

Do not solve one problem.

Create a solution that can solve hundreds.

Reusable thinking creates scalable companies.

---

# Principle 11

The report is the product.

Everything else exists to improve reports.

Landing pages.

Animations.

Settings.

Templates.

All are secondary.

If reports improve...

ResearchAI improves.

---

# Principle 12

Evidence beats opinion.

Ideas are cheap.

Evidence creates products.

Every important decision should be supported by:

User behavior.

Feedback.

Testing.

Measurement.

---

# Principle 13

Build for ten years.

Every important decision should answer:

Will this still make sense in ten years?

If not...

Think again.

---

# Principle 14

AI is replaceable.

ResearchAI is not.

Models will change.

Providers will change.

Technology will evolve.

Our philosophy, user experience and trust must remain.

ResearchAI should be valuable regardless of which AI model powers it.

---

# Principle 15

One mission.

One direction.

Every document.

Every feature.

Every line of code.

Every design decision.

Every release.

Must support one mission:

Help people complete meaningful work faster.

Nothing else matters more.

---

# The ResearchAI Question

Before making any decision ask one question:

Does this make the user's work meaningfully easier?

If the answer is yes...

Continue.

If the answer is no...

Stop.

---

# Final Statement

ResearchAI is not a website.

ResearchAI is not an AI wrapper.

ResearchAI is not a collection of features.

ResearchAI is a long-term system designed to help humanity transform questions into professional work.

Everything we build must deserve to exist.
