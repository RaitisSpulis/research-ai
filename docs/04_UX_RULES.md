# UX Rules

## Purpose

This document defines how every interaction inside ResearchAI should feel.

Users should never need to learn how to use ResearchAI.

The interface should feel natural from the first second.

If users have to think about the interface...

We have failed.

---

# Rule 1

## The Five Second Rule

Within five seconds of opening the website the user must understand:

• What this product does.

• What problem it solves.

• What they should do next.

If these three questions are not answered...

The landing page has failed.

---

# Rule 2

## One Primary Action

Every page has one main goal.

Landing:

Start research.

Dashboard:

Continue work.

Report:

Read and act.

Settings:

Control the experience.

Never create multiple competing primary actions.

---

# Rule 3

## Never Leave The User Stuck

Every screen must answer:

"What should I do now?"

After every completed action there must be a logical next step.

Example:

Report completed.

↓

Export

↓

Create Presentation

↓

Generate Business Plan

↓

Start another report

Never end with a dead screen.

---

# Rule 4

## Every Click Gives Feedback

Buttons must always react.

Forms must always respond.

Loading must always communicate progress.

Users should never wonder:

"Did it work?"

---

# Rule 5

## Loading Should Build Confidence

Loading screens should explain what is happening.

Example:

Reading information...

Analyzing sources...

Building report...

Generating recommendations...

Almost finished...

Users wait more patiently when they understand the process.

---

# Rule 6

## Empty States Must Teach

An empty page should never feel empty.

Instead it should explain:

What this page does.

Why it matters.

How to start.

Every empty state should contain one clear action.

---

# Rule 7

## Errors Should Help

Never display technical errors.

Never blame the user.

Instead explain:

What happened.

What they can do.

How to recover.

Every error should include a solution.

---

# Rule 8

## The User Should Feel Progress

Research should feel alive.

Progress bars.

Step indicators.

Animated sections.

Completion percentages.

Everything should communicate movement.

---

# Rule 9

## Reduce Decision Fatigue

Never show ten choices when three are enough.

Highlight the recommended action.

Hide advanced controls until needed.

Simple first.

Advanced later.

---

# Rule 10

## Reports Must Be Scannable

Nobody wants giant walls of text.

Reports should use:

Cards.

Tables.

Charts.

Lists.

Highlights.

Summaries.

Recommendations.

Users should understand the report in under two minutes.

---

# Rule 11

## Every Report Ends With Action

Information without action creates uncertainty.

Every report must answer:

What should I do next?

Actionable recommendations are mandatory.

---

# Rule 12

## Never Waste Attention

Attention is limited.

Only show information that helps the current task.

Decorations should never compete with content.

---

# Rule 13

## Mobile First Thinking

Every feature should work naturally on phones.

No horizontal scrolling.

No tiny buttons.

No hidden actions.

Touch interactions must feel effortless.

---

# Rule 14

## Keyboard Friendly

Professional users work quickly.

Support keyboard shortcuts.

Tab navigation.

Escape to close dialogs.

Enter to submit.

Accessibility improves productivity.

---

# Rule 15

## Speed Feels Like Quality

Fast interfaces feel smarter.

Animations should support perception.

Never slow users down.

Performance is part of UX.

---

# Rule 16

## Consistency Creates Trust

The same interaction should always produce the same result.

Users should build confidence over time.

Never surprise them.

---

# Rule 17

## Every Page Must Feel Alive

Subtle animations.

Hover feedback.

Smooth transitions.

Skeleton loading.

Micro interactions.

The interface should feel responsive.

Not distracting.

Alive.

---

# Rule 18

## Build Momentum

After every success...

Encourage the next useful action.

Example:

Research completed.

↓

Create presentation.

↓

Export PDF.

↓

Start another report.

Momentum increases engagement.

---

# Rule 19

## Make Professional Work Feel Easy

The user should feel like they completed hours of work...

Even though they only spent minutes.

That feeling is the core UX of ResearchAI.

---

# Final Rule

The interface is invisible.

The user's success is visible.

If users remember the interface...

We designed too much.

If users remember what they accomplished...

We designed correctly.
